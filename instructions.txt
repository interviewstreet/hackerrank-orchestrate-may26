==========================================================================
HackerRank Orchestrate Support Triage Agent — Manual Run Instructions
==========================================================================

This file tells an evaluator how to install, run, and manually verify the
agent in this repository. Estimated time end-to-end: ~10 minutes.

The agent reads support_tickets/support_tickets.csv, processes every
ticket through (preprocessor -> classifier -> retriever -> reasoner ->
grounding verifier -> escalation decision -> output writer), and writes
support_tickets/output.csv.

--------------------------------------------------------------------------
1. PREREQUISITES
--------------------------------------------------------------------------

  - Python 3.11 or newer (3.10 may work; 3.11+ is what we tested).
  - Internet access (the agent calls the Anthropic API).
  - An Anthropic API key with access to claude-sonnet-4-5.
  - ~700 MB of free disk (model weights + corpus index).

A virtual environment is strongly recommended.

--------------------------------------------------------------------------
2. INSTALL
--------------------------------------------------------------------------

  Step 1: enter the repo (you are presumably already here).

      cd hackerrank-orchestrate-may26

  Step 2: create + activate a virtual environment.

    macOS / Linux (bash, zsh):
      python3 -m venv .venv
      source .venv/bin/activate

    Windows (PowerShell):
      py -3.11 -m venv .venv
      .\.venv\Scripts\Activate.ps1

  Step 3: install dependencies.

      pip install --upgrade pip
      pip install -r code/requirements.txt

  First install pulls sentence-transformers + torch + faiss-cpu and is the
  slowest step (~3-5 minutes on a normal connection).

--------------------------------------------------------------------------
3. CONFIGURE THE .env FILE
--------------------------------------------------------------------------

  The agent reads ANTHROPIC_API_KEY from a .env file at the repo root
  (loaded automatically via python-dotenv). A template is provided.

    macOS / Linux:
      cp .env.example .env

    Windows (PowerShell):
      Copy-Item .env.example .env

  Open .env in a text editor and replace the placeholder with your real
  key:

      ANTHROPIC_API_KEY=sk-ant-...your-real-key-here...

  The other variables in .env are optional overrides for tunable knobs
  (RETRIEVAL_MIN_SCORE, etc.) and can stay at their defaults.

  The .env file is in .gitignore and will not be committed.

--------------------------------------------------------------------------
4. CORPUS INDEX (already built, rebuild only if data/ changes)
--------------------------------------------------------------------------

  This repo ships with a pre-built retrieval index at code/index/:

      chunks.parquet     ~3.8 MB
      faiss.index        ~21 MB
      bm25.pkl           ~6.5 MB
      manifest.json      ~130 KB

  If you change anything under data/ (e.g. add or edit corpus articles),
  rebuild the index:

      python code/indexer.py --rebuild

  Or rebuild as part of the main run:

      python code/main.py --rebuild-index

  The first build downloads BAAI/bge-small-en-v1.5 model weights from
  Hugging Face (~33 MB, one-time) and embeds ~13 400 corpus chunks. On
  CPU this takes 5-15 minutes.

--------------------------------------------------------------------------
5. RUN A SMALL SAMPLE FIRST (RECOMMENDED)
--------------------------------------------------------------------------

  Before processing the full input, run on the 10-row sample CSV to
  confirm everything is wired up. 3 tickets is enough for a sanity check:

      python code/main.py --input support_tickets/sample_support_tickets.csv --output support_tickets/output.sample.csv --limit 3

  Expected wall-clock: 30-60 seconds (the first ticket is slowest because
  the embedding model loads). Expected cost: ~$0.005-0.01 per ticket on
  claude-sonnet-4-5.

  You should see output like:

      [main] loading retriever from .../code/index ...
      [main] loading tickets from support_tickets/sample_support_tickets.csv ...
      [main] processing 3 ticket(s) ...
      [main]   1/3   Replied req=product_issue   pa=screen          (29.3s)
      [main]   2/3 Escalated req=bug             pa=general_support (4.8s)
      [main]   3/3   Replied req=product_issue   pa=library         (15.9s)
      [main] done in 50.0s -> support_tickets/output.sample.csv (replied=2, escalated=1, errors=0)

--------------------------------------------------------------------------
6. RUN THE FULL INPUT
--------------------------------------------------------------------------

      python code/main.py

  Defaults:
      --input         support_tickets/support_tickets.csv
      --output        support_tickets/output.csv
      --config        code/config.yaml
      --corpus-root   data
      --index-dir     code/index

  Useful flags:
      --limit N         process only the first N rows (for quick checks)
      --start N         skip the first N rows (for resume)
      --rebuild-index   force corpus reindex before running

  Expected wall-clock: ~5-10 minutes for ~30 tickets. Expected cost:
  ~$0.20-0.30 for the full run on claude-sonnet-4-5.

--------------------------------------------------------------------------
7. HOW TO VERIFY THE OUTPUT MANUALLY
--------------------------------------------------------------------------

  (a) The output file exists at support_tickets/output.csv.

  (b) Header is exactly (lowercase, comma-separated, no spaces):

      issue,subject,company,status,product_area,response,justification,request_type

    macOS / Linux:
      head -n 1 support_tickets/output.csv

    Windows (PowerShell):
      Get-Content support_tickets/output.csv -TotalCount 1

  (c) Row count matches the input row count.

    macOS / Linux:
      diff <(wc -l < support_tickets/support_tickets.csv) <(wc -l < support_tickets/output.csv)

    Windows (PowerShell):
      (Get-Content support_tickets/support_tickets.csv).Count - (Get-Content support_tickets/output.csv).Count

    Both files include a header row, so the line counts should match.

  (d) Casing rules (eyeball a few rows):

      - status        is exactly "Replied" or "Escalated" (TitleCase)
      - request_type  is one of {product_issue, feature_request, bug, invalid} (lowercase)
      - product_area  is lowercase snake_case (e.g. screen, claude_api_and_console, fraud_protection)

  (e) Spot-check 2-3 Replied rows: the response should be a coherent
      answer that references something documented under data/. For
      example, a HackerRank Screen question should produce a response
      grounded in data/hackerrank/screen/*.md.

  (f) Spot-check 2-3 Escalated rows: the justification should name a
      trigger T-1..T-6 (e.g. "trigger T-3: outage / service-down report;
      escalated for engineering follow-up.").

  (g) No Replied row should contain a fabricated phone number, URL, or
      dollar amount that is not present in some data/ corpus file. The
      grounding verifier (code/verifier.py) blocks these at runtime by
      flipping the row to Escalated with trigger T-1.

--------------------------------------------------------------------------
8. RUN THE TEST SUITE
--------------------------------------------------------------------------

  110 unit + integration tests (loader, output writer, indexer,
  retriever, preprocessor, classifier heuristics, classifier schema with
  mocked Anthropic, reasoner with mocked Anthropic, verifier). All tests
  use mocked LLM calls and require no API key.

      cd code
      pytest -q

  Expected: 110 passed. Wall-clock ~90-110 seconds (the retriever tests
  build a tiny fixture index via sentence-transformers).

  Run a single fast subset (excludes the retriever fixture build):

      pytest -q --ignore=tests/test_retriever.py --ignore=tests/test_indexer.py

--------------------------------------------------------------------------
9. COMMON ISSUES
--------------------------------------------------------------------------

  - "ANTHROPIC_API_KEY is not set"
        Confirm .env exists at the repo root, contains
        ANTHROPIC_API_KEY=sk-ant-..., and that you are running from the
        repo root (not from inside code/). Restart your shell after
        editing .env if your editor wrote a stale buffer.

  - "Index dir does not exist"
        Build the index first:
            python code/indexer.py --rebuild
        Or pass --rebuild-index to main.py to do it inline.

  - 401 / authentication_error from the Anthropic API
        The API key is invalid, revoked, or lacks access to
        claude-sonnet-4-5. Verify in https://console.anthropic.com.

  - 404 model_not_found
        The pinned model claude-sonnet-4-5 is no longer available on
        your account. Edit code/config.yaml and code/classifier.py /
        code/reasoner.py to point at claude-sonnet-4-6 or the latest
        Sonnet model your account supports.

  - Slow first ticket (~30s)
        Expected. The sentence-transformers model loads on the first
        retrieval call. Subsequent tickets are 5-15 seconds each.

  - "FAILED ... NotImplementedError" in pytest
        You are running pytest against an older checkout. Pull latest
        and reinstall requirements.

--------------------------------------------------------------------------
10. COST ESTIMATE (claude-sonnet-4-5)
--------------------------------------------------------------------------

  Per ticket:
      classifier  ~300-500 input tokens, ~150 output -> ~$0.001
      reasoner    ~2000-4000 input tokens, ~400 output -> ~$0.005-0.01
      Total per ticket: ~$0.006-0.011

  Full input (~30 tickets): ~$0.20-0.35.
  Sample CSV (10 tickets):  ~$0.05-0.10.

  Embeddings are computed locally via BAAI/bge-small-en-v1.5 and cost
  nothing.
